# 5. Персистентность данных на Render

Файловая система контейнера **временная**: без диска индекс и загрузки пропадают при redeploy.

## Persistent Disk

1. Render → сервис **LightRAG** → **Disks** → добавить диск (например **10 GB**).
2. Задать **Mount path**, например **`/var/data`** (путь потом менять нельзя без миграции данных).

## Переменные путей

Префикс должен совпадать с mount path диска:

| Переменная | Для mount `/var/data` |
|------------|------------------------|
| `WORKING_DIR` | `/var/data/rag_storage` |
| `INPUT_DIR` | `/var/data/inputs` |

Если mount path другой (например `/mnt/data`), замените префикс: `/mnt/data/rag_storage` и `/mnt/data/inputs`.

## Ограничения Render

- С диском нет **zero-downtime** deploy.
- Один инстанс (нельзя горизонтально масштабировать один диск).
- Сохраняется только содержимое **под** mount path.

## После первого подключения диска

Том может быть пустым — документы нужно загрузить заново (или восстановить из бэкапа).

## Альтернатива

Внешняя БД (PostgreSQL, MongoDB и т.д.) — см. `env.example` в апстриме; сложнее в настройке.
